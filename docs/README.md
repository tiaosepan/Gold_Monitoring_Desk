## 📝 文档维护规范

### 文档分类原则

1. **design/** - 系统设计、架构、技术方案、分析报告
2. **guides/** - 使用教程、操作指南、配置说明
3. **issues/** - 问题记录、Bug修复、调试过程
4. **reports/** - 审计报告、状态报告、变更记录
5. **archive/** - 已过期的文档、历史版本
6. **assets/** - 截图、配置文件、示例数据

### 文档命名规范

- 使用清晰的中文名称
- 添加日期后缀（如有必要）：`文档名_YYYY-MM-DD.md`
- 使用下划线分隔：`使用指南_新功能.md`

### 归档原则

文档应归档到 `archive/` 当：
- 文档描述的问题已解决且不再相关
- 文档被新版本替代
- 文档涉及的功能已被重构或移除
- 文档内容已过期超过 30 天
