## 🐛 已知问题

### 问题1: V1和V2并行运行的影响

**现象**: 如果同时启用V1和V2，数据库会同时记录两种版本的数据

**影响**: 数据量增加，但不影响功能

**解决**: 
- 选择一个版本使用
- 通过 `note` 字段区分版本（V2包含`[V2]`标记）

---

### 问题2: 切换版本后历史数据无法迁移

**现象**: V1和V2数据结构相同，但判定逻辑不同

**影响**: 历史数据等级值无法自动转换

**解决**:
- 保留历史数据，不做迁移
- 通过 `note` 字段区分版本
- 新数据使用新版本逻辑

### 常见问题

**Q: 切换版本后没有效果？**

A: 需要重启服务。切换版本只修改数据库配置，需要重启才能加载新版本。

```bash
# 停止服务
Ctrl+C

# 重新启动
python backend/main.py
```

---

**Q: 如何验证优化是否生效？**

A: 检查以下三点：

1. **RSS上下文分析**: 查看日志 `logs/rss.log`，确认有"检测到负面词组"等日志
2. **美债数据源**: 查询 `/api/us10y/latest`，确认 `source` 字段为 `CNBC` 等新数据源
3. **反转等级V2**: 查询 `/api/reversal/latest`，确认 `note` 字段包含 `[V2]` 标记

---

**Q: V2加权版和简单版哪个更好？**

A: 
- **简单版**: 适合大多数场景，逻辑清晰，易于理解
- **加权版**: 适合对准确率要求高的场景，考虑信号强度，更精确

建议先使用简单版，运行稳定后再考虑加权版。
