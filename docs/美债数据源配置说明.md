## 二、自动降级策略

系统采用 **快速失败策略**，优先使用实时数据源，失败后自动降级：

```python
async def fetch_and_save_data(self, tenor: str = '10y'):
    """
    降级顺序: CNBC → Sina → FRED → Eastmoney → Yahoo
    
    每个数据源都有独立的超时控制，确保快速失败
    """
    
    # 1️⃣ 尝试CNBC (2秒超时)
    try:
        yield_value = await asyncio.wait_for(self.cnbc_api.get_yield(tenor), timeout=2.0)
        if yield_value > 0:
            return yield_value, 'CNBC'
    except:
        pass
    
    # 2️⃣ 尝试Sina (2秒超时)
    try:
        yield_value = await asyncio.wait_for(self.api.get_us10y_data(), timeout=2.0)
        if yield_value > 0:
            return yield_value, 'Sina'
    except:
        pass
    
    # 3️⃣ 尝试FRED (3秒超时)
    if self.fred_api_key:
        try:
            yield_value = await asyncio.wait_for(self._fetch_from_fred(tenor), timeout=3.0)
            if yield_value > 0:
                return yield_value, 'FRED'
        except:
            pass
    
    # 4️⃣ 尝试Eastmoney (3秒超时)
    try:
        yield_value = await asyncio.wait_for(eastmoney_api.get_yield(tenor), timeout=3.0)
        if yield_value > 0:
            return yield_value, 'Eastmoney'
    except:
        pass
    
    # 5️⃣ 最后尝试Yahoo (5秒超时)
    try:
        yield_value = await asyncio.wait_for(self._fetch_from_yahoo(tenor), timeout=5.0)
        if yield_value > 0:
            return yield_value, 'Yahoo'
    except:
        pass
    
    # ❌ 所有数据源都失败
    raise ValueError("所有数据源都无法获取有效数据")
```

## 三、配置方法

### 方法1: 默认配置（推荐）✅

**无需任何配置**，系统会自动使用CNBC和Sina数据源。

```bash
# 直接启动即可
python backend/main.py
```

**适用场景**: 大多数情况下，CNBC和Sina足够使用

---

### 方法2: 启用FRED官方数据（可选）

如需使用FRED官方数据源，需要配置API密钥：

#### 步骤1: 获取FRED API密钥

访问 https://fred.stlouisfed.org/docs/api/api_key.html 注册并获取免费API密钥

#### 步骤2: 配置环境变量

**Windows PowerShell:**
```powershell
# 临时设置（当前会话有效）
$env:FRED_API_KEY="your_api_key_here"

# 永久设置（用户级）
[System.Environment]::SetEnvironmentVariable("FRED_API_KEY", "your_api_key_here", "User")
```

**Windows CMD:**
```cmd
set FRED_API_KEY=your_api_key_here
```

**Linux/Mac:**
```bash
export FRED_API_KEY="your_api_key_here"

# 永久设置（添加到 ~/.bashrc 或 ~/.zshrc）
echo 'export FRED_API_KEY="your_api_key_here"' >> ~/.bashrc
source ~/.bashrc
```

#### 步骤3: 重启服务

```bash
# 停止服务
Ctrl+C

# 重新启动
python backend/main.py
```

### 问题3: FRED API返回错误

**错误信息**:
```
FRED API错误: Invalid API key
```

**解决方案**:

1. **验证API密钥是否正确**:
```powershell
# 检查环境变量
echo $env:FRED_API_KEY  # Windows
echo $FRED_API_KEY      # Linux/Mac
```

2. **测试API密钥**:
```bash
curl "https://api.stlouisfed.org/fred/series/observations?series_id=DGS10&api_key=YOUR_KEY&file_type=json&limit=1"
```

3. **如果密钥无效，重新获取**:
   - 访问 https://fred.stlouisfed.org/
   - 登录账号
   - 前往 My Account > API Keys
   - 生成新密钥

### 1. 生产环境配置

```bash
# 推荐配置FRED API作为备用
export FRED_API_KEY="your_api_key"

# 配置合理的超时时间
export US10Y_TIMEOUT=10  # 总超时10秒
```

### 2. 高可用部署

```yaml
# docker-compose.yml
services:
  app:
    environment:
      - FRED_API_KEY=${FRED_API_KEY}
      - US10Y_SOURCES=CNBC,Sina,FRED  # 指定使用的数据源
    restart: always
    
  # 添加健康检查
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/api/us10y/latest"]
      interval: 30s
      timeout: 5s
      retries: 3
```

### 3. 告警配置

```yaml
# prometheus_alerts.yml
groups:
  - name: us10y_alerts
    rules:
      - alert: US10YDataSourceFailed
        expr: rate(us10y_fetch_total{status="failed"}[5m]) > 0.5
        for: 5m
        annotations:
          summary: "美债数据源失败率过高"
          description: "{{ $labels.source }} 数据源5分钟内失败率 {{ $value }}%"
```
