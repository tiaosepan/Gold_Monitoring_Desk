## 🔍 日志系统

### 日志文件位置
所有日志文件存储在 `logs/` 目录下：

```
logs/
├── main.log              # 主程序日志（启动/关闭）
├── main_error.log        # 主程序错误日志
├── scheduler.log         # 调度器日志（任务执行）
├── scheduler_error.log   # 调度器错误日志
├── sge_monitor.log       # SGE监控日志
├── sge_monitor_error.log # SGE监控错误日志
├── reversal_detector.log # 反转检测日志
├── rss_collector.log     # RSS采集日志
├── us10y_monitor.log     # 美债监控日志
└── api.log               # API请求日志
```

### 查看日志

**Windows PowerShell**:
```powershell
# 查看最新日志
Get-Content logs\scheduler.log -Tail 50

# 实时监控日志
Get-Content logs\scheduler.log -Wait -Tail 20

# 查看错误日志
Get-Content logs\scheduler_error.log
```

**Linux/Mac**:
```bash
# 查看最新日志
tail -f logs/scheduler.log

# 搜索错误
grep ERROR logs/*.log
```

### 日志级别说明

- **DEBUG**: 详细执行信息（如：任务开始/结束）
- **INFO**: 正常操作信息（如：数据采集成功）
- **WARNING**: 警告信息（如：数据源异常）
- **ERROR**: 错误信息（如：任务执行失败）

## 📊 Prometheus监控

### 快速启动

#### 1. 查看指标端点
启动系统后，访问:
```
http://localhost:8000/metrics
```

你会看到类似输出:
```
# HELP sge_fetch_total SGE数据采集总次数
# TYPE sge_fetch_total counter
sge_fetch_total{status="success"} 145.0
sge_fetch_total{status="failed"} 2.0

# HELP sge_premium_cny_per_g 当前SGE溢价（元/克）
# TYPE sge_premium_cny_per_g gauge
sge_premium_cny_per_g 3.8989
```

#### 2. 安装Prometheus（可选）

**Windows**:
1. 下载: https://prometheus.io/download/
2. 解压到 `C:\prometheus`
3. 复制配置: 
   ```powershell
   copy monitoring\prometheus.yml C:\prometheus\
   copy monitoring\alert_rules.yml C:\prometheus\
   ```
4. 启动:
   ```powershell
   cd C:\prometheus
   .\prometheus.exe --config.file=prometheus.yml
   ```
5. 访问: http://localhost:9090

#### 3. 安装Grafana（可选）

**Windows**:
1. 下载: https://grafana.com/grafana/download
2. 解压并启动:
   ```powershell
   .\bin\grafana-server.exe
   ```
3. 访问: http://localhost:3000（admin/admin）
4. 添加Prometheus数据源（URL: http://localhost:9090）
5. 导入仪表盘: `monitoring/grafana_dashboard.json`

### 常用查询

在Prometheus UI中尝试以下查询:

```promql
# SGE采集频率（每分钟）
rate(sge_fetch_total[1m]) * 60

# SGE采集成功率
rate(sge_fetch_total{status="success"}[5m]) / rate(sge_fetch_total[5m]) * 100

# API P95响应时间
histogram_quantile(0.95, rate(api_response_duration_seconds_bucket[5m]))

# 当前反转等级
reversal_signal_level

# RSS高分事件数
rss_high_score_events_total
```

## 🐛 故障排查

### 问题1: 日志文件未生成

**检查**:
1. 确认 `logs/` 目录存在
2. 检查写入权限
3. 查看控制台输出

**解决**:
```bash
mkdir logs
```

### 问题2: Prometheus指标无数据

**检查**:
1. 访问 http://localhost:8000/metrics
2. 确认系统已启动
3. 等待至少1个采集周期（60秒）

### 问题3: 测试失败

**检查**:
1. 确认已安装测试依赖:
   ```bash
   pip install pytest pytest-asyncio
   ```
2. 检查Python版本（需要3.11+）
3. 查看详细错误:
   ```bash
   pytest tests/ -v -s
   ```

### 问题4: 美债数据全为0

**原因**: 新浪API美债数据不稳定

**解决方案**:
1. **推荐**: 配置FRED API
   ```powershell
   $env:FRED_API_KEY="your_api_key_here"
   ```
   获取免费密钥: https://fred.stlouisfed.org/docs/api/api_key.html

2. **备选**: 使用东方财富API（已集成，但不稳定）

### 问题5: RSS源采集失败

**检查**:
```bash
python scripts\utils\check_data_sources.py
```

**处理异常源**:
```bash
python scripts\utils\disable_bad_rss_sources.py
```

## 📝 配置优化建议

### 1. 配置FRED API（推荐）

**步骤**:
1. 访问 https://fredaccount.stlouisfed.org/login/secure/
2. 注册账号并获取API密钥
3. 设置环境变量:
   ```powershell
   # PowerShell（临时）
   $env:FRED_API_KEY="your_key_here"
   
   # PowerShell（永久）
   [System.Environment]::SetEnvironmentVariable("FRED_API_KEY", "your_key_here", "User")
   ```

4. 重启系统验证:
   ```bash
   python scripts\utils\check_data_sources.py
   ```

### 2. 优化日志配置

编辑 `backend/utils/logger.py` 可调整:
- 日志级别（默认: DEBUG）
- 文件大小限制（默认: 10MB）
- 备份数量（默认: 5个）

### 3. 调整监控参数

编辑 `monitoring/prometheus.yml`:
```yaml
scrape_interval: 15s  # 抓取间隔（可调整为30s/60s）
scrape_timeout: 10s   # 抓取超时
```

## 📞 技术支持

### 常见问题

**Q: 日志文件太大怎么办？**  
A: 系统自动轮转（10MB限制），旧日志自动压缩。可手动删除 `*.log.1`、`*.log.2` 等备份文件。

**Q: Prometheus占用内存过高？**  
A: 调整 `monitoring/prometheus.yml` 的 `scrape_interval`，从15秒改为30秒或60秒。

**Q: 测试失败怎么办？**  
A: 查看详细错误 `pytest tests/ -v -s`，确认数据库文件权限和Python依赖版本。

**Q: 美债数据一直为0？**  
A: 配置FRED API密钥（推荐）或等待新浪API恢复。
