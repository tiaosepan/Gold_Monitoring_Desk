## ✅ 已修复：时间显示问题

### 问题描述

用户看到"最新采集的文章是昨天晚上的 2026/3/25 21:35:24"，误以为系统没有更新。

### 根本原因

RSS事件有两个时间：
- **采集时间（fetched_at）**: 系统抓取文章的时间（今天13:44）
- **发布时间（published_at）**: RSS源发布文章的时间（昨天21:35）

前端只显示"发布时间"，导致用户误解。

### 修复方案

**修改文件**: `static/app.js` 第563行

**修改前**:
```javascript
<span class="tag">${formatTime(item.published_at || item.fetched_at)}</span>
```

**修改后**:
```javascript
<span class="tag">📥 采集: ${formatTime(item.fetched_at)}</span>
<span class="tag">📰 发布: ${formatTime(item.published_at || item.fetched_at)}</span>
```

### 效果对比

**修改前**:
```
[政治缓和] [虎嗅网] [昨天 21:35]
```
❌ 看起来像旧新闻

**修改后**:
```
[政治缓和] [虎嗅网] [📥 采集: 今天 13:44] [📰 发布: 昨天 21:35]
```
✅ 清楚显示是"刚采集的新闻"

### 验证

刷新浏览器后，RSS事件流将同时显示：
- 📥 采集时间 - 显示系统何时获取到文章（最新）
- 📰 发布时间 - 显示文章的原始发布时间

## ⚠️ 待优化：标题和摘要中文翻译

### 问题描述

### 现状对比

❌ Philippines Suspends Electricity Market to Prevent Price Surge
❌ US and Iran Wrangle Over Talks, No End in Sight
```

#### 原系统（中文为主）
```
