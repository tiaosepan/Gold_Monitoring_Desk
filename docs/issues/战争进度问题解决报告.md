## 解决方案

### 修改事件分类逻辑

**文件**: `backend/services/rss_collector.py`

**修改位置**: `classify_event` 方法（第107-120行）

**修改内容**:

```python
# 修改前：直接使用源分类
def classify_event(self, title: str, summary: str, source_category: str) -> str:
    return source_category

# 修改后：根据内容动态判断
def classify_event(self, title: str, summary: str, source_category: str) -> str:
    """分类RSS事件（完全匹配原系统逻辑）"""
    content = f"{title} {summary}".lower()
    
    # 匹配战争关键词
    war_matches = []
    for category in ['positive', 'negative', 'uncertain']:
        for keyword in self.WAR_KEYWORDS[category]:
            if keyword.lower() in content:
                war_matches.append(keyword)
    
    # 匹配政治关键词
    political_matches = []
    for category in ['positive', 'negative', 'uncertain']:
        for keyword in self.POLITICAL_KEYWORDS[category]:
            if keyword.lower() in content:
                political_matches.append(keyword)
    
    # 如果有战争关键词，优先归类为war（匹配原系统行为）
    if war_matches:
        return 'war'
    elif political_matches:
        return 'political'
    else:
        return source_category
```

**核心改进**：
1. ✅ 不再仅依赖RSS源的category
2. ✅ 根据文章内容动态匹配关键词
3. ✅ 优先判断为war类型（匹配原系统逻辑）
4. ✅ 完全匹配原系统的分类行为
