# X 轴显示格式修正

**修正时间**: 2026-03-28  
**问题**: 零点日期显示格式与原系统不一致

---

## 问题描述

### 用户截图显示

从用户提供的截图可以看到，原系统的 X 轴显示格式为：

```
22:00  23:00  28  01:00  02:00  03:00  04:00
              ↑
          只显示日期数字（28）
```

### 修改前

复刻系统的显示：
```
22:00  23:00  03-28  01:00  02:00  03:00  04:00
              ↑
        显示完整的月-日格式（03-28）
```

**差异**：零点位置显示 `03-28` 而不是 `28`

---

## 修正方案

### 修改前代码

```javascript
formatter: function(value) {
  const date = new Date(value);
  const hours = date.getHours();
  const minutes = date.getMinutes();
  
  // 如果是00:00，显示完整日期
  if (hours === 0 && minutes === 0) {
    return `${String(date.getMonth() + 1).padStart(2, '0')}-${String(date.getDate()).padStart(2, '0')}`;
  }
  
  // 否则显示时间
  return `${String(hours).padStart(2, '0')}:00`;
}
```

**输出示例**：
```
22:00  23:00  03-28  01:00  02:00
```

### 修改后代码

```javascript
formatter: function(value) {
  const date = new Date(value);
  const hours = date.getHours();
  const minutes = date.getMinutes();
  
  // 如果是00:00，只显示日期的"日"部分
  if (hours === 0 && minutes === 0) {
    return String(date.getDate());  // ← 关键修改
  }
  
  // 否则显示时间
  return `${String(hours).padStart(2, '0')}:00`;
}
```

**输出示例**：
```
22:00  23:00  28  01:00  02:00
```

## 设计理念

### 为什么只显示日期数字？

1. **节省空间**：`28` 比 `03-28` 更简洁
2. **视觉清晰**：日期数字作为"日界标记"更醒目
3. **国际惯例**：许多金融图表在跨日时只显示日期数字

### 用户如何理解？

- **时间标签**（`HH:00`）：小时刻度
- **日期数字**（`DD`）：日界标记
- 前后时间递增 → 用户能推断日期含义

例如：
```
22:00  23:00  28  01:00  02:00
       ↑      ↑    ↑
    3月27日  3月28日零点  3月28日凌晨1点
```

## 技术实现

### JavaScript Date API

```javascript
const date = new Date(value);

// 获取日期数字（1-31）
date.getDate()           // 返回: 28

// 获取月份（0-11，需要+1）
date.getMonth() + 1      // 返回: 3

// 获取小时（0-23）
date.getHours()          // 返回: 0, 1, 2, ..., 23
```

### 判断逻辑

```javascript
if (hours === 0 && minutes === 0) {
  // 零点位置
  return String(date.getDate());
} else {
  // 其他时间
  return `${hours}:00`;
}
```
