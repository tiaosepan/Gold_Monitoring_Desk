# 调度器错误修复报告

**修复时间**: 2026-03-26  
**问题来源**: 定时任务执行失败（RSS采集和反转检测）

---

## 问题诊断

### 错误1: RSS采集任务失败
```
RSS采集任务执行失败: 'category'
```

**根本原因**: 
- 调度器 `scheduler.py` 第144行使用了 `event['category']`
- 但 `RSSCollectorService.get_high_score_events()` 方法返回的字段是 `event_type`，不是 `category`
- 同时缺少 `score`、`source_name` 和 `link` 字段

### 错误2: 反转检测任务失败
```
反转检测任务执行失败: 'reversal_level'
```

**根本原因**:
- 调度器 `scheduler.py` 第179行使用了 `result['reversal_level']`
- 但 `ReversalDetectorService.detect_and_save()` 返回的结构是 `result['data']['signal_level']`

---

## 修复方案

### 修复1: 完善 `get_high_score_events` 返回字段

**文件**: `backend/services/rss_collector.py` (第527-535行)

**修改前**:
```python
return [{
    'id': r.id,
    'fetched_at': r.fetched_at.isoformat(),
    'title': r.title,
    'event_type': r.event_type,
    'impact_score': r.impact_score,
    'impact_level': r.impact_level,
    'matched_keywords': r.matched_keywords
} for r in records]
```

**修改后**:
```python
return [{
    'id': r.id,
    'fetched_at': r.fetched_at.isoformat(),
    'title': r.title,
    'event_type': r.event_type,
    'category': r.event_type,  # 别名，用于调度器
    'impact_score': r.impact_score,
    'score': r.impact_score,  # 别名，用于调度器
    'impact_level': r.impact_level,
    'matched_keywords': r.matched_keywords,
    'source_name': r.source.name if r.source else 'Unknown',
    'link': r.link
} for r in records]
```

**改进点**:
- ✅ 添加 `category` 字段（`event_type` 的别名）
- ✅ 添加 `score` 字段（`impact_score` 的别名）
- ✅ 添加 `source_name` 字段（从关联的 `source` 对象获取）
- ✅ 添加 `link` 字段（事件链接）

### 修复2: 修正调度器中的反转检测逻辑

**文件**: `backend/scheduler.py` (第178-183行)

**修改前**:
```python
# 如果需要推送，创建并发送警报
if result['success'] and result['reversal_level'] >= 3:
    alert = service.create_alert(result['reversal_level'], result['conditions'])
    if alert:
        await self._send_alert(alert)
```

**修改后**:
```python
# 如果需要推送，创建并发送警报
if result['success'] and result.get('data'):
    signal_level = result['data'].get('signal_level', 0)
    if signal_level >= 2:  # Level 2及以上才推送
        alert = service.create_alert(result['data'])
        if alert:
            await self._send_alert(alert)
```

**改进点**:
- ✅ 正确访问 `result['data']['signal_level']` 而不是 `result['reversal_level']`
- ✅ 传递完整的 `data` 字典给 `create_alert()` 方法
- ✅ 调整推送阈值为 Level 2（匹配原系统逻辑）

## 技术细节

### 字段映射关系
| 调度器期望字段 | 服务返回字段 | 解决方案 |
|--------------|------------|---------|
| `category` | `event_type` | 添加别名字段 |
| `score` | `impact_score` | 添加别名字段 |
| `source_name` | 无 | 从 `r.source.name` 获取 |
| `link` | 无 | 从 `r.link` 获取 |
| `result['reversal_level']` | `result['data']['signal_level']` | 修正访问路径 |

### 推送逻辑
- **RSS事件推送**: 评分 ≥ 7 的事件会被推送
- **反转警报推送**: Level ≥ 2 的反转信号会被推送（Level 3和4只记录不推送）
