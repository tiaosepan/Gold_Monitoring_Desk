# RSS时间显示完整说明

**问题**: 为什么最新采集的文章显示的是"昨天晚上的时间"？  
**解答时间**: 2026-03-26 14:30

---

## 🔑 核心概念

### RSS事件有两个时间

#### 1. 采集时间（fetched_at）
- **含义**: 我们的系统从RSS源抓取文章的时间
- **示例**: 2026-03-26 14:14:43（今天下午2点14分）
- **特点**: 这是"新鲜"的时间，表示系统刚刚获取到这条新闻

#### 2. 发布时间（published_at）
- **含义**: RSS源网站发布文章的时间
- **示例**: 2026-03-25 21:35:24（昨天晚上9点35分）
- **特点**: 这是"原始"的时间，表示文章是什么时候写的

## 📊 实际数据示例

### 最新事件（ID:109）

```
标题: 产能损毁固化远期溢价，谈判仍难平抑原油上冲风险
来源: 虎嗅网

【采集时间】2026-03-26 13:44:51 ← 今天下午采集的（新！）
【发布时间】2026-03-25 21:35:24 ← 昨天晚上发布的（旧）

时间差: 16.2小时
```

### 为什么有16小时的时间差？

**这是RSS的正常工作机制！**

```
21:35 (昨天) - 虎嗅网发布文章
21:40 (昨天) - RSS feed更新
...
13:44 (今天) - 我们的系统采集到（15分钟轮询周期）
```

RSS不是实时推送系统，而是**定期轮询**机制：
1. 新闻网站发布文章后，RSS feed才会更新
2. 我们的系统每15分钟轮询一次RSS feed
3. 所以从"发布"到"采集"会有延迟

## 🖥️ 前端显示的是哪个时间？

### 当前实现（app.js 第563行）

```javascript
<span class="tag">${formatTime(item.published_at || item.fetched_at)}</span>
```

**显示的是：发布时间（published_at）**

### 为什么这样设计？

**优点**：
- ✅ 显示新闻的真实发布时间
- ✅ 用户知道新闻是什么时候写的
- ✅ 符合新闻阅读习惯

**缺点**：
- ❌ 看起来像"旧新闻"（昨天的时间）
- ❌ 用户可能误以为系统没有更新
- ❌ 无法直观看出"刚刚采集到"

### Q: 为什么最新采集的文章是昨天晚上的？

**A: 因为前端显示的是"发布时间"，而不是"采集时间"！**

**真相**：
- ✅ 文章是**今天下午13:44采集的**（最新！）
- ✅ 但文章是**昨天晚上21:35发布的**（RSS源的发布时间）

**这是正常的！** RSS源的文章通常不是实时发布的，可能是几小时甚至几天前写的。
