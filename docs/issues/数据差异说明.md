# SGE 溢价走势 vs 黄金反转监控 - 数据差异说明

**创建时间**: 2026-03-28  
**问题描述**: 用户质疑这两个图表为什么数据不一样

---

## 问题分析

### 用户看到的现象

从截图观察：
1. **左图**（SGE 溢价走势）：绿色曲线约 900-1050 范围
2. **右图**（黄金反转监控）：青绿色曲线约 4000-4500 范围
3. 两个图表的曲线**形状相似但数值差异巨大**

### 原因：这是正常的设计！

---

## 📊 两个图表的数据来源

### 左图：SGE 溢价走势（`renderSgeChart`）

**数据表**: `sge_prices`

**显示内容**:
```javascript
{
  人民币金价: items[index].shfe_price_cny_per_g,      // 元/克
  国际金折算: items[index].london_price_cny_per_g,    // 元/克
  溢价:       items[index].premium_cny_per_g           // 元/克
}
```

**数据来源**:
- 沪金连续 (nf_AU0) - 新浪财经 API
- 伦敦金 (hf_XAU) - 新浪财经 API
- 汇率 (USDCNY) - 新浪财经 API

**Y 轴单位**: **元/克**（人民币）

**典型值**: 900-1200 元/克

---

### 右图：黄金反转监控（`renderReversalChart`）

**数据表**: `reversal_conditions`

**显示内容**:
```javascript
{
  现货金价格: items[index].gold_price_usd_per_oz,     // 美元/盎司
  反转信号:   mapSignalToY(items[index].signal_level)  // 等级映射
}
```

**数据来源**:
- 伦敦现货金 (hf_XAU) - 新浪财经 API
- RSS 事件、MA15、美债收益率综合判断

**Y 轴单位**: **美元/盎司**（USD/oz）

**典型值**: 4000-5000 美元/盎司

## 🔢 单位换算验证

### 换算公式

```
人民币/克 = (美元/盎司 × 汇率) ÷ 31.1035
```

其中：
- 1 盎司 = 31.1035 克（金衡盎司）
- 汇率约 7.2（USDCNY）

### 实际验证

假设当前数据：
- 右图显示：**4200 美元/盎司**
- 当前汇率：**7.2**

计算左图应显示的值：
```
4200 × 7.2 ÷ 31.1035 = 972.36 元/克
```

**对比截图**：
- 左图绿色曲线约在 **970 元/克** 附近 ✅
- 右图青绿曲线约在 **4200 美元/盎司** 附近 ✅

**结论**: 数据**完全一致**，只是单位不同！

---

## 📝 为什么要用两个不同单位？

### 左图（SGE）- 关注国内市场

**目的**: 监控**上海金交所溢价**
- 需要比较沪金和国际金的**人民币价格**
- 溢价 = 沪金 - 国际金（都是元/克）
- 单位统一为**元/克**方便直观对比

**用户关心**: 
> 国内金价比国际贵多少？溢价是否异常？

### 右图（反转）- 关注国际趋势

**目的**: 检测**全球金价反转信号**
- 使用国际标准单位**美元/盎司**
- 结合 MA15、RSS 事件、美债收益率
- 评估反转等级（1-4级）

**用户关心**:
> 国际金价趋势如何？是否有反转信号？

## ✅ 总结

| 项目 | SGE 溢价走势 | 黄金反转监控 |
|------|--------------|--------------|
| **数据表** | `sge_prices` | `reversal_conditions` |
| **金价单位** | 元/克 | 美元/盎司 |
| **数据源** | 沪金 + 伦敦金 | 伦敦现货金 |
| **主要目的** | 监控国内溢价 | 检测反转信号 |
| **典型值** | 900-1200 | 4000-5000 |
| **更新频率** | 60秒 | 反转检测周期 |
| **Y轴左侧** | 人民币金价 | 现货金价格 |
| **Y轴右侧** | 溢价（元/克） | 反转等级（1-4） |

### 关键结论

1. ✅ **数据本质相同** - 都是金价，只是单位不同
2. ✅ **展示目的不同** - 一个看溢价，一个看趋势
3. ✅ **曲线形状相似** - 因为追踪的是同一市场
4. ✅ **数值验证通过** - 换算后完全吻合

**这不是 bug，而是符合系统设计的正常表现！**

## 🛠️ X 轴显示问题（已修复）

虽然数据差异是正常的，但在用户的截图中发现了真正的问题：

**X 轴显示混乱**：
```
21:00  22:00  23:00  28  01:00  02:00  03:00  04:00
                    ↑ 这个"28"（日期）混在时间标签中
```

**根本原因**：
- 黄金反转监控图的 `xAxis` 配置中**缺少** `hideOverlap: true` 和 `minorTick: { show: false }`
- 导致 ECharts 自动插入日期标签且无法智能隐藏重叠标签

**修复方法**：
```javascript
xAxis: { 
  type: "time",
  axisLabel: { 
    hideOverlap: true,  // ✅ 防止标签重叠
    formatter: function(value) {
      // ... 格式化逻辑
    }
  },
  minorTick: { show: false }  // ✅ 隐藏次级刻度
}
```

**修复后效果**：
```
20:00  22:00  03-28  02:00  04:00
              ↑ 日期只在零点显示
```
