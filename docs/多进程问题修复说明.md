# 多进程问题修复说明

## 问题描述

之前系统出现严重的多进程问题：
- 同时运行了6个Python后端实例
- 每个实例都有自己的调度器，导致任务重复执行
- 反转检测任务应该每60秒执行一次，但实际每7-8秒执行一次
- 数据库中积累了大量异常高频数据：
  - **反转数据**: 81.2%的数据间隔<30秒，有52次0秒间隔
  - **SGE数据**: 86.7%的数据是异常高频
  - **美债数据**: 92.8%的数据是异常高频，平均间隔仅7秒

## 已实施的修复措施

### 1. 数据清理

运行清理脚本清除异常数据：
```bash
python clean_high_frequency_data.py
```

清理结果：
- 反转数据：284条→97条（清理65.8%）
- SGE数据：1557条→766条（清理50.8%）
- 美债数据：10052条→2524条（清理74.9%）
- RSS事件：759条→115条（清理84.8%）

### 2. 进程锁机制

新增 `backend/utils/process_lock.py`：
- 使用文件锁防止多实例运行
- 启动时检测现有进程，拒绝重复启动
- 退出时自动释放锁

### 3. 安全启动脚本

新增 `safe_start.py`：
- 自动检测并终止旧进程
- 清理锁文件
- 启动新实例
- 提供详细的启动日志

### 4. 配置修改

修改 `backend/main.py`：
- 禁用uvicorn热重载（`reload=False`）
- 集成进程锁检查
- 启动时验证唯一性

## 正确的启动方式

### 推荐方式（自动防护）

```bash

```

### 不推荐方式（无防护）

## 手动检查进程

### Windows

```powershell
# 查找所有Python进程
Get-Process python

# 停止所有Python进程
Stop-Process -Name python -Force

# 停止特定PID
Stop-Process -Id <PID> -Force
```

### Linux/Mac

```bash
# 查找后端进程
ps aux | grep main.py

# 停止进程
kill <PID>

# 强制停止
kill -9 <PID>
```

## 预防措施

1. **始终使用启动脚本**
   - 使用`start.bat`或`safe_start.py`
   - 避免直接运行`python main.py`

2. **重启前检查旧进程**
   - 安全启动脚本会自动处理
   - 手动重启前先用`Get-Process python`检查

3. **监控数据采集频率**
   - 正常频率：反转60秒、SGE 60秒、美债60秒、RSS 900秒
   - 异常频率：<30秒间隔、0秒间隔
   - 定期运行`check_high_frequency_data.py`检查

4. **锁文件位置**
   - 位置：`Gold_Monitoring_Desk/backend.lock`
   - 内容：运行中的进程PID
   - 异常时可手动删除（确认无进程后）

## 数据库健康检查

定期运行检查脚本：
```bash
python check_high_frequency_data.py
```

如发现异常数据，运行清理脚本：
```bash
python clean_high_frequency_data.py
```

## 故障排除

### 问题：启动时提示"后端已在运行"，但实际没有运行

解决方案：
```bash
# 1. 确认确实没有进程
Get-Process python

# 2. 删除锁文件
del backend.lock

# 3. 重新启动
.\start.bat
```

### 问题：数据采集频率异常

解决方案：
```bash
# 1. 检查数据频率
python check_high_frequency_data.py

# 2. 检查运行进程数
Get-Process python

# 3. 如有多个进程，全部停止
Stop-Process -Name python -Force

# 4. 清理异常数据
python clean_high_frequency_data.py

# 5. 使用安全启动
.\start.bat
```
