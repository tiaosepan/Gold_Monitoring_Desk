### 🔧 问题2: RSS事件流显示旧内容

**错误表现**:
- 数据库中有最新事件（ID:78-82，采集11:38）
- 但前端显示的是旧事件（ID:51，采集09:27）

**根本原因**: API使用了错误的排序字段（`published_at` 而不是 `fetched_at`）

**修复文件**: `backend/api/routes.py`

#### 修复位置1: `/api/reversal/events` (第662-665行)

```python
# 修改前
events = query.order_by(
    RSSEvent.published_at.desc()
).limit(limit).all()

# 修改后
events = query.order_by(
    RSSEvent.fetched_at.desc(),
    RSSEvent.published_at.desc()
).limit(limit).all()
```

#### 修复位置2: `/api/reversal/status` (第526-529行)

```python
# 修改前
recent_events = db.query(RSSEvent).order_by(
    RSSEvent.published_at.desc()
).limit(50).all()

# 修改后
recent_events = db.query(RSSEvent).order_by(
    RSSEvent.fetched_at.desc(),
    RSSEvent.published_at.desc()
).limit(50).all()
```

**详见**: `RSS更新问题诊断报告.md`

### 🔧 问题4: RSS抓取记录和反转运行记录缺失

**错误**: `/api/reversal/status` 中的 `recent_rss_fetch_runs` 和 `recent_runs` 都是空列表

**修复文件**: 
- `backend/api/routes.py` (第530-558行)
- `backend/services/reversal_detector.py` (第275行后、第295行后)

#### 修复4.1: 填充RSS抓取记录

```python
# 修改前
recent_rss_fetch_runs = []

# 修改后
rss_runs = db.query(UpdateRecord).filter_by(
    data_type='rss_events'
).order_by(UpdateRecord.created_at.desc()).limit(10).all()

recent_rss_fetch_runs = [{
```
