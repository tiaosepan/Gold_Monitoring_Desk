# 智能采集策略说明

**实施时间**: 2026-03-28  
**优化目标**: 根据市场开盘状态动态调整采集频率，节省资源

---

## 用户需求

> "数据休市了，是不是可以不用频繁采集，等开市了再来采集？"

**分析**：
- 市场休市时，价格不变
- 频繁采集相同数据浪费资源
- 应该根据市场状态动态调整频率

---

## 实施方案

### 智能采集策略

根据市场开盘状态自动调整采集频率：

| 市场状态 | 采集频率 | 说明 |
|---------|---------|------|
| **双市场开盘** | 60秒/次 | 沪金+伦敦金都开盘，价格活跃 |
| **单市场开盘** | 120秒/次 | 仅一个市场开盘，价格变化较慢 |
| **全部休市** | 300秒/次 | 双市场都休市，价格不变 |

### 市场时间定义

**沪金市场**（周一至周五）：
- 日盘：09:00 - 15:30
- 夜盘：21:00 - 02:30

**伦敦金市场**：
- 周一至周五：24小时交易
- 周末：休市

## 技术实现

### 1. 市场状态检测

**文件**：`backend/services/sge_monitor.py`

**函数**：`_check_market_open()`

```python
def _check_market_open(self) -> Dict[str, int]:
    now = datetime.now()
    hour = now.hour
    minute = now.minute
    weekday = now.weekday()  # 0=周一, 6=周日
    
    # 沪金市场：周一至周五 9:00-15:30 和 21:00-02:30
    shfe_open = 0
    if weekday < 5:
        # 日盘
        if (hour == 9 and minute >= 0) or (9 < hour < 15) or (hour == 15 and minute <= 30):
            shfe_open = 1
        # 夜盘
        elif hour >= 21 or hour < 2 or (hour == 2 and minute <= 30):
            shfe_open = 1
    
    # 伦敦金市场：周一到周五全天
    london_open = 1 if weekday < 5 else 0
    
    # 双市场都开盘
    both_open = 1 if shfe_open == 1 and london_open == 1 else 0
    
    return {
        'shfe_market_open': shfe_open,
        'london_market_open': london_open,
        'both_markets_open': both_open
    }
```

### 3. 动态调整频率

**文件**：`backend/scheduler.py`

**函数**：`sge_monitor_task()`

```python
async def sge_monitor_task(self):
    async with SGEMonitorService(db) as service:
        result = await service.fetch_and_save_data()
        
        # 检查市场状态，动态调整采集频率
        if result.get('market_status'):
            both_open = result['market_status'].get('both_markets_open', 0)
            shfe_open = result['market_status'].get('shfe_market_open', 0)
            
            # 根据市场状态决定采集频率
            if both_open == 1:
                interval_seconds = 60      # 双市场开盘
            elif shfe_open == 1:
                interval_seconds = 120     # 单市场开盘
            else:
                interval_seconds = 300     # 全部休市
            
            # 重新调度任务
            self.scheduler.reschedule_job(
                'sge_monitor',
                trigger=IntervalTrigger(seconds=interval_seconds)
            )
```

## 注意事项

### 1. 首次运行

首次启动时默认为60秒间隔，执行一次采集后才会根据市场状态调整。

### 2. 时区问题

系统使用本地时间（CST），确保服务器时区设置正确。

### 3. 节假日

当前逻辑未处理节假日，交易日节假日会判定为"工作日"继续采集。

可优化方案：
```python
# 可以加入节假日判断
holidays = ['2026-01-01', '2026-02-10', ...]
if today_str in holidays:
    return {'shfe_market_open': 0, ...}
```

### 4. 手动触发

即使在休市期间，用户仍可通过API手动触发采集：
```
POST /api/sge/collect
```
