# 采集频率优化说明

**优化时间**: 2026-03-28  
**优化目标**: 提高黄金反转监控图表的曲线平滑度

---

## 问题背景

用户反馈两个图表的曲线不一样：
- **SGE 溢价走势**：曲线平滑连续
- **黄金反转监控**：曲线粗糙，有明显阶梯感

**根本原因**：采集频率差异
- SGE：每 60 秒采集 → 24小时约 1440 个数据点
- 反转：每 10 分钟检测 → 24小时约 144 个数据点

---

| **SGE监控** | 60秒 | **60秒** ✓ | ~1440 | 平滑 |
| **美债监控** | 60秒 | **60秒** ✓ | ~1440 | 平滑 |
| **反转检测** | 10分钟 | **1分钟** ⚡ | ~1440 | **平滑** ✅ |
| **RSS采集** | 15分钟 | **15分钟** ✓ | ~96 | 无需图表 |

## 性能影响评估

### 数据库写入量

| 任务 | 修改前 | 修改后 | 增幅 |
|------|--------|--------|------|
| **SGE监控** | 1440次/天 | 1440次/天 | - |
| **反转检测** | 144次/天 | **1440次/天** | **+900%** ⚡ |
| **总写入** | 1584次/天 | 2880次/天 | +82% |

### 计算开销

**每次反转检测需要**：
1. 查询最近60分钟 SGE 数据（~60条）
2. 计算 MA15 移动平均
3. 查询最近3小时 RSS 事件（政治/战争）
4. 查询最近24小时美债数据
5. 综合判断信号等级

**评估**：
- ✅ 查询量：每次仅查询少量记录，索引优化后影响小
- ✅ 计算量：简单算术运算，耗时<10ms
- ✅ 网络请求：无（仅查本地数据库）
- ⚠️ 磁盘 I/O：增加 10 倍写入（SQLite 能承受）

## 优化建议

### 1. 数据库索引优化

确保以下索引存在（提高查询速度）：

```sql
-- SGE 价格表
CREATE INDEX IF NOT EXISTS idx_sge_fetched_at 
ON sge_prices(fetched_at DESC);

-- RSS 事件表
CREATE INDEX IF NOT EXISTS idx_rss_fetched_type_score 
ON rss_events(fetched_at, event_type, impact_score);

-- 美债数据表
CREATE INDEX IF NOT EXISTS idx_us10y_fetched_at 
ON us_treasury(fetched_at DESC);

-- 反转条件表
CREATE INDEX IF NOT EXISTS idx_reversal_fetched_at 
ON reversal_conditions(fetched_at DESC);
```

### 2. 数据清理策略

随着频率提高，建议定期清理旧数据：

```python
# 示例：保留最近30天数据
def cleanup_old_data():
    cutoff = datetime.now() - timedelta(days=30)
    db.query(ReversalCondition).filter(
        ReversalCondition.fetched_at < cutoff
    ).delete()
    db.commit()
```

### 3. 内存缓存优化
