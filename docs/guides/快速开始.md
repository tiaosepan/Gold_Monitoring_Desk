# 黄金监控中台 - 快速开始指南

## 一、系统启动

### 1. 启动服务器

**Windows用户**：
```bash
# 双击运行
start.bat

# 或在命令行中运行
cd d:\www\prototype\Gold_Monitoring_Desk
start.bat
```

**Linux/Mac用户**：
```bash
cd d:\www\prototype\Gold_Monitoring_Desk/backend
python main.py
```

### 2. 访问系统

启动成功后，打开浏览器访问：
```
http://localhost:8000
```

## 二、首次使用配置

### 1. 配置钉钉推送（可选）

如果需要接收钉钉推送通知，需要配置钉钉机器人：

**方法1：通过API配置**
```bash
# 更新Webhook URL
curl -X PUT http://localhost:8000/api/config/dingtalk_webhook \
  -H "Content-Type: application/json" \
  -d "https://oapi.dingtalk.com/robot/send?access_token=YOUR_TOKEN"

# 更新Secret
curl -X PUT http://localhost:8000/api/config/dingtalk_secret \
  -H "Content-Type: application/json" \
  -d "YOUR_SECRET"
```

**方法2：直接修改数据库**
```sql
-- 使用SQLite工具打开 database/gold_monitor.db
UPDATE push_targets 
SET webhook_url = 'https://oapi.dingtalk.com/robot/send?access_token=YOUR_TOKEN',
    secret = 'YOUR_SECRET'
WHERE id = 1;
```

### 2. 调整监控阈值（可选）

默认阈值：
- SGE溢价阈值：15元/克
- SGE溢价百分比阈值：0.5%
- 美债下跌阈值：0.1%
- RSS政治事件阈值：7分
- RSS战争事件阈值：7分

如需调整，可通过API修改：
```bash
# 修改SGE溢价阈值为20元/克
curl -X PUT http://localhost:8000/api/config/sge_premium_threshold \
  -H "Content-Type: application/json" \
  -d "20"
```

## 七、常见问题

### Q1: 启动后无法访问？

**A**: 检查端口8000是否被占用：
```bash
# Windows
netstat -ano | findstr :8000

# Linux/Mac
lsof -i :8000
```

如果被占用，可以修改 `backend/main.py` 中的端口号。

### Q2: 数据采集失败？

**A**: 可能的原因：
1. 网络连接问题 - 检查网络是否正常
2. 新浪API限流 - 等待一段时间后重试
3. API代码变更 - 查看错误日志

查看日志：
```bash
# 查看系统状态
curl http://localhost:8000/api/system/status
```

### Q3: RSS采集返回404？

**A**: 部分RSS源URL可能已失效，这是正常的。可以：
1. 更新RSS源URL
2. 禁用失效的源
3. 添加新的RSS源

### Q4: 钉钉推送不工作？

**A**: 检查：
1. Webhook URL是否正确
2. Secret是否配置
3. 钉钉机器人是否启用
4. 查看推送日志：
```bash
curl http://localhost:8000/api/system/status
```

### Q5: 如何停止服务？

**A**: 
- 在命令行窗口按 `Ctrl+C`
- 或直接关闭命令行窗口

### Q6: 如何备份数据？

**A**: 直接复制数据库文件：
```bash
copy database\gold_monitor.db database\gold_monitor_backup.db
```

### Q7: 如何清空数据重新开始？

**A**: 
```bash
# 删除数据库文件
del database\gold_monitor.db

# 重新初始化
cd backend
..\venv\Scripts\python database.py
```

## 八、性能优化建议

### 1. 数据库优化

定期清理历史数据：
```sql
-- 删除30天前的SGE数据
DELETE FROM sge_prices WHERE created_at < datetime('now', '-30 days');

-- 删除30天前的RSS事件
DELETE FROM rss_events WHERE created_at < datetime('now', '-30 days');
```

### 2. 调整采集频率

如果系统负载过高，可以降低采集频率。编辑 `backend/scheduler.py`：

```python
# 将SGE监控从5分钟改为10分钟
self.scheduler.add_job(
    self.sge_monitor_task,
    trigger=IntervalTrigger(minutes=10),  # 原来是5
    ...
)
```

### 3. 限制数据量

在API调用时限制返回数量：
```bash
# 只获取最近50条数据
curl http://localhost:8000/api/sge/latest?limit=50
```
