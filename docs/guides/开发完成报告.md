## 已知问题和解决方案

### 1. 新浪API数据代码问题 ✅ 已解决
**问题**: 初始使用的数据代码不正确，导致获取的数据异常。

**解决方案**: 
- 将SGE代码从 `hf_GC` 改为 `gds_AU9999`（沪金99）
- 将国际金价代码从 `DINIW` 改为 `hf_XAU`（伦敦金现货）
- 将汇率代码从 `fx_susdcny` 改为 `USDCNY`

### 2. 新浪API 403错误 ✅ 已解决
**问题**: 直接请求新浪API返回403 Forbidden。

**解决方案**: 添加User-Agent和Referer请求头。

### 3. 数据库初始化问题 ✅ 已解决
**问题**: SQL脚本执行失败，初始数据未插入。

**解决方案**: 改用SQLAlchemy ORM直接插入初始数据。

### 4. 美债数据源问题 ⚠️ 待完善
**问题**: 新浪API的美债代码返回空数据。

**建议**: 需要找到正确的美债数据代码或使用FRED API作为备用数据源。

### 5. 部分RSS源404 ⚠️ 正常
**问题**: 人民网和央视网的RSS源返回404。

**说明**: 这些网站可能已经更改了RSS地址，需要更新URL或移除失效源。

编辑数据库 `push_targets` 表或通过API更新：

```sql
UPDATE push_targets 
SET webhook_url = 'https://oapi.dingtalk.com/robot/send?access_token=YOUR_TOKEN',
    secret = 'YOUR_SECRET'
WHERE id = 1;
```

### 系统参数配置
可通过API或直接修改 `system_config` 表：

```bash
# 通过API更新
curl -X PUT http://localhost:8000/api/config/sge_premium_threshold \
  -H "Content-Type: application/json" \
  -d "20"
```
