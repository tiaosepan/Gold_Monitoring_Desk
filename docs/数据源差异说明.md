# SGE 人民币金价数据源差异说明

## 问题描述

参考网站 http://rev.sccit.com.cn:8000/ 与我们系统显示的 **人民币金价** 存在差异。

## 差异原因

### 我们的系统
- **数据源**：新浪财经 API
- **人民币金价**：`nf_AU0`（**沪金连续期货合约**）
- **国际金价**：`hf_XAU`（伦敦金现货，美元/盎司）
- **汇率**：`USDCNY`

### 参考网站
- 可能使用上海黄金交易所官方 API
- 可能使用 `AU9999`（上海金现货）或 `Au(T+D)`（黄金延期交收合约）

### 核心差异
1. **期货 vs 现货**
   - 我们使用的 `nf_AU0` 是**期货合约**
   - 参考网站可能使用的是**现货价格**
   - 期货价格包含升水/贴水，与现货通常有 0.5-2 元/克的差异

2. **数据更新频率**
   - 新浪财经 API：可能有 1-5 秒延迟
   - 交易所官方 API：更接近实时

3. **交易成本**
   - 不同合约的交易费用、保证金率不同
   - 导致市场报价存在细微差异

## 已实施的解决方案

### 方案：在页面明确标注数据源

#### 1. 前端图表标题更新
- **位置**：`static/index.html`
- **修改**：在图表标题处添加 "（沪金连续期货 nf_AU0）" 标注
- **效果**：用户可以清楚知道我们使用的是期货数据

```html
<!-- 总览页面 -->
<h3>SGE 溢价走势</h3>
<p class="muted">人民币金价（沪金连续期货 nf_AU0）、国际金折算与溢价曲线</p>

<!-- 详情页面 -->
<h3>SGE 溢价图表</h3>
<p class="muted">人民币金价（沪金连续期货 nf_AU0）- 用于看盘和确认相对国际金的溢价环境</p>
```

#### 2. 图表图例更新
- **位置**：`static/app.js`
- **修改**：将图例从 "人民币金价" 改为 "人民币金价(期货)"
- **效果**：图表中直接显示数据类型

```javascript
legend: { 
  top: 6, 
  textStyle: { color: "#5f6b7c" }, 
  data: ["溢价", "人民币金价(期货)", "国际金折算"]

```

#### 3. 后端数据源说明更新
- **位置**：`backend/services/sge_monitor.py`
- **修改**：在 note 字段中明确标注 "期货"
- **效果**：在最近溢价记录表格的状态列中显示数据源信息

```python

# 休市时
note = "国内源: nf_AU0 期货休市返回0，使用历史均值代理；国际源: 新浪 hf_XAU 伦敦金现货"

# 盘后
note = "国内源: nf_AU0 期货盘后价(参考)；国际源: 新浪 hf_XAU 伦敦金现货"
```

## 其他可选方案（未实施）

### 方案 A：切换到上海金现货数据
- **优点**：与参考网站数据一致
- **缺点**：
  - 需要接入上海黄金交易所官方 API（可能需要申请权限）
  - 开发成本较高
  - 新浪财经的 `gds_AU9999` 更新频率较低

### 方案 B：同时显示期货和现货价格
- **优点**：数据更全面
- **缺点**：页面复杂度增加，用户可能混淆

### 方案 C：添加数据源切换功能
- **优点**：用户可以自由选择
- **缺点**：开发成本高，维护复杂

## 数据准确性说明

虽然与参考网站的数据不完全相同，但我们的数据具有以下优势：

1. **数据来源可靠**：新浪财经是主流金融数据提供商
2. **更新稳定**：`nf_AU0` 沪金连续合约数据更新稳定，不受单一合约到期影响
3. **溢价计算准确**：期货-现货溢价本身就是市场关注的重要指标
4. **实时性好**：期货市场流动性强，价格反应更快

## 建议

1. **当前方案已足够**：通过标注数据源，用户可以理解价格差异的原因
2. **关注溢价趋势**：溢价的变化趋势比绝对值更重要
3. **如需精确现货价格**：可以参考上海黄金交易所官网 https://www.sge.com.cn/

## 技术细节

### 数据流程
```
新浪财经 API
  ↓
nf_AU0 沪金连续期货价格 (元/克) → shfe_price_cny_per_g
hf_XAU 伦敦金现货 (美元/盎司) → london_price_usd_per_oz
USDCNY 汇率 → usdcny_rate
  ↓
国际金折算 (元/克) = london_price_usd_per_oz / 31.1035 × usdcny_rate
  ↓
溢价 (元/克) = shfe_price_cny_per_g - london_price_cny_per_g
```

### 单位换算
```python
# 盎司转克（1盎司 = 31.1035 克）
def oz_to_gram(oz_price: float) -> float:
    return oz_price / 31.1035
```
