## 🎯 优化 #1: RSS上下文分析增强

### 问题描述

**原问题**: 关键词匹配缺少上下文分析，导致误判

**典型误判案例**:
```python
# ❌ 原算法问题
text = "停火失败，冲突继续"
# 匹配到"停火" → 正面+1分
# 匹配到"冲突" → 负面-1分
# 结果: 中性，但实际应该是明显负面

text = "未能达成和平协议"
# 匹配到"和平"和"协议" → 正面+2分
# 实际: 这是失败的谈判，应该是负面
```

---

### 解决方案

#### 1.1 负面词组检测（优先级最高）

```python
# 新增: 负面词组库
NEGATIVE_PHRASES = {
    'political': [
        '停火失败', '谈判破裂', '谈判失败', '外交破裂', 
        '协议作废', '协议失效',
        'ceasefire failed', 'talks collapsed', 
        'negotiations broke down'
    ],
    'war': [
        '停战失败', '和平破裂', '撤军失败', 
        '航道关闭', 'peace failed', 
        'withdrawal failed'
    ]
}

# 检测逻辑
for phrase in negative_phrases:
    if phrase in text:
        return [f"[负面词组]{phrase}"], 5, -3  # 直接扣3分
```

✅ **效果**: "停火失败"会被正确识别为负面

---

#### 1.2 否定上下文检测

```python
# 新增: 否定词库
NEGATION_WORDS = [
    '不', '未', '没', '无', '非',
    'not', 'no', 'never', 'without', 'failed', 'collapse'
]

# 检测逻辑
for negation in NEGATION_WORDS:
    for positive_keyword in keywords['positive']:
        if f"{negation}{positive_keyword}" in text:
            return [f"[否定]{negation}+{positive_keyword}"], 5, -2
```

✅ **效果**: "未能停火"、"没有和平"会被正确识别为负面

---

#### 1.3 关键词权重系统

```python
# 新增: 关键词权重（重要性不同）
KEYWORD_WEIGHTS = {
    # 高权重关键词（3.0）
    '停火': 3.0,
    'ceasefire': 3.0,
    '轰炸': 3.0,
    'bombing': 3.0,
    
    # 中权重关键词（2.0-2.5）
    '谈判': 2.5,
    'talks': 2.5,
    '制裁': 2.5,
    '袭击': 2.5,
    
    # 低权重关键词（1.5）
    '对话': 1.5,
    '讨论': 1.5
}

# 评分计算
positive_score = sum(KEYWORD_WEIGHTS.get(kw, 1.0) for kw in positive_keywords)
adjustment += int(positive_score)  # 加权加分
```

✅ **效果**: "停火"比"对话"重要3倍

#### 2.1 5个数据源自动降级

系统现已支持 **5个美债数据源**:

| 优先级 | 数据源 | 实时性 | 可靠性 | 超时 |
|-------|--------|--------|--------|------|
| 🥇 1 | **CNBC** | ⚡ 实时 | ⭐⭐⭐⭐⭐ | 2秒 |
| 🥈 2 | **Sina Finance** | ⚡ 实时 | ⭐⭐⭐⭐ | 2秒 |
| 🥉 3 | **FRED** | 📅 每日 | ⭐⭐⭐⭐⭐ | 3秒 |
| 4 | **Eastmoney** | ⚡ 实时 | ⭐⭐⭐ | 3秒 |
| 5 | **Yahoo Finance** | ⚡ 实时 | ⭐⭐⭐ | 5秒 |

#### 2.2 快速失败策略

```python
async def fetch_and_save_data(self, tenor='10y'):
    """
    快速失败策略: 每个数据源独立超时
    """
    # 1️⃣ CNBC (2秒超时)
    try:
        yield_value = await asyncio.wait_for(
            self.cnbc_api.get_yield(tenor), 
            timeout=2.0
        )
        if yield_value > 0:
            return yield_value, 'CNBC'
    except:
        pass  # 快速失败，尝试下一个
    
    # 2️⃣ Sina (2秒超时)
    try:
        yield_value = await asyncio.wait_for(
            self.api.get_us10y_data(), 
            timeout=2.0
        )
        if yield_value > 0:
            return yield_value, 'Sina'
    except:
        pass
    
    # 3️⃣ FRED (3秒超时)
    # 4️⃣ Eastmoney (3秒超时)
    # 5️⃣ Yahoo (5秒超时)
    ...
```

## 🎯 优化 #3: 反转等级判断优化

### 问题描述

**原问题**: 等级定义混乱，用户难以理解

**混淆点**:
1. Level 4 = "仅政治信号" = "最弱"？
2. Level 1 = "4个信号全触发" = "最强"？
3. 推送策略: Level 1/2推送，Level 3/4不推送？

❌ **不符合直觉**: Level应该是数字越大越重要

---

### 解决方案

#### 3.1 V2简单计数法（推荐）

**核心思想**: 等级 = 信号数量

```python
def determine_reversal_level_v2(...):
    """
    Level 0: 0个信号 - 无反转迹象
    Level 1: 1个信号 - 弱信号，观察阶段
    Level 2: 2个信号 - 中等信号，关注阶段
    Level 3: 3个信号 - 强信号，准备行动 [推送]
    Level 4: 4个信号 - 极强信号，立即行动 [推送]
    """
    signal_count = price_signal + political_signal + war_signal + us10y_signal
    return signal_count
```

✅ **优点**: 
- 直观易懂
- Level 4 > Level 1（符合直觉）
- 推送策略清晰（Level 3/4推送）

---

#### 3.2 V2加权评分法（高级）

**核心思想**: 考虑信号强度，更精确

```python
def determine_reversal_level_weighted(...):
    """
    加权评分系统（总分100）
    
    评分规则:
    - 价格信号: 25分 × 信号强度(0-1)
    - 政治信号: 25分 × 信号强度(0-1)
    - 战争信号: 25分 × 信号强度(0-1)
    - 美债信号: 25分 × 信号强度(0-1)
    
    等级转换:
    - 0-20分: Level 0
    - 21-40分: Level 1
    - 41-60分: Level 2
    - 61-80分: Level 3 [推送]
    - 81-100分: Level 4 [推送]
    """
    weighted_score = sum(weight * signal * strength 
                         for each signal)
    return score_to_level(weighted_score)
```

**信号强度示例**:
```python
# 价格反弹3% vs 阈值1.2% → 强度 = 3/1.2 = 2.5 (限制为2.0)
# RSS事件评分8分 → 强度 = (8-5)/5 = 0.6
# 美债下跌10bp vs 阈值1bp → 强度 = 10/1 = 10.0 (限制为2.0)
```

✅ **优点**: 更精确，考虑信号质量
